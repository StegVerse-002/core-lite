# StegVerse Ecosystem Invariants

These are canonical. They apply to every component, every stage, every deployment.

## 1. Memory Is Not Authority
Stored context may inform a transition, but only an admissibility check under the
active stage can authorize consequence-bearing action.

## 2. Context Is Not Consent
A file, note, screenshot, or prompt existing in memory does not automatically
authorize downstream use.

## 3. Capability Is Not Admissibility
A system being capable of performing an action does not make the action allowed.

## 4. Input Is Not Execution
Submitting an input does not mean the system may execute consequence-bearing
actions from it.

## 5. LLM Reasoning Is Not Authorization
External AI output may inform a transition but cannot authorize action by itself.

## 6. Identity Context Is Not Identity Proof
KnowledgeVault may preserve identity context, but it does not verify legal,
biometric, institutional, or account identity unless a separate verification
mechanism is declared.

## 7. Local Success Is Not Composite Success
A local fix or component success does not prove the full transition is successful.

## 8. Every Consequence-Bearing Transition Needs a Receipt
Any transition that affects state, authority, memory use, or execution must
produce evidence.

## 9. Every Stage Needs a STOP Condition
No stage is open-ended.

## 10. Unknown Fails to Review or Fail-Closed
Unknown input, unknown authority, unknown privacy, or unknown consequence
must not be silently allowed.
