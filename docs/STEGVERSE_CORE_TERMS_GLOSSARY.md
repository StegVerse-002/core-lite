# StegVerse Core Terms Glossary

| Term | Definition |
|------|------------|
| **admissibility** | Whether a proposed action is allowed under the current stage, scope, and legitimacy checks |
| **actor** | The entity performing an action (e.g. StegVerse-002, human_operator) |
| **bundle** | A packaged set of files with a manifest, hashes, and declared scope |
| **candidate** | A proposed file, patch, or change prepared for operator review — not yet applied |
| **CGE** | Continuation/Governance/Execution — the admissibility decision engine |
| **context packet** | A bounded, declared selection of KnowledgeVault memory for a specific AI use |
| **gate** | An active transition boundary with a declared STOP condition |
| **governed tier** | A capability level earned through legitimacy checks, not claimed |
| **identity context** | User-declared continuity information — not identity proof |
| **ingestion** | The process of receiving and classifying an input through CGE |
| **input** | Any submitted content: text, voice, image, screenshot, document, LLM output, bundle |
| **instruction packet** | A bounded transition candidate derived from LLM output via the Adapter Gate |
| **intake** | The full orchestrated run: CGE fingerprint → admissibility → receipts → outputs |
| **KnowledgeVault** | User-owned portable memory substrate — not AI memory |
| **legitimacy check** | Evidence-based proof required to advance a stage |
| **LLM output** | External AI reasoning — reasoning evidence, not authority |
| **manifest** | A machine-readable description of an input, bundle, or context packet |
| **memory** | Stored vault content — not authorization for action |
| **mutation** | A change to a file, config, or state |
| **receipt** | Append-only evidence of a consequence-bearing transition |
| **reconstruction** | The ability to replay how a state came to exist from receipts |
| **scope** | The declared boundary within which an action is permitted |
| **stage** | A defined position in the Transition Table with allowed/forbidden actions and STOP |
| **STOP condition** | The declared endpoint for a gate — no open-ended stages |
| **tier** | A deployment level in the Governed Tiered Intellect Model |
| **transition** | A move from one stage to the next, requiring legitimacy checks |
| **Transition Table** | The capability staging model for consequence-bearing intelligence |
| **TVC** | Token Vault Core — runtime package authority and scoped execution token issuer |
| **TV** | Token Vault — policy and secret distribution hub |
| **vault** | The KnowledgeVault folder structure owned by the user |
